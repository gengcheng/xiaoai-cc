# xiaoai-cc

> 对小爱音箱说一句话，让电脑上的 **Claude Code** 来回答，再用**小爱自己的声音**念出来。

一个node小程序：你对小爱说带触发词的话，本应用通过小米官方云读到这句话、交给 `claude` 处理，再把答案下发成 TTS 让小爱读出来。不依赖 Docker、数据库或浏览器后台，电脑和音箱**不用在同一局域网**，同一个小米账号、各自能上网即可。

```
你对小爱说话 ──► 小米云存下对话记录
   ──► 本应用每秒轮询 ──► 命中触发词 ──► claude -p "<问题>"
      ──► 调小米云 TTS ──► 小爱念出答案
```

## 效果

对小爱说：

> **「问问克劳德，太阳系最大的行星是哪个」**

几秒后，小爱用它自己的声音回答：

> 太阳系里最大的行星是木星，它的体积比其它七颗行星加起来还大……

- **普通问答**：触发词后面的话直接当问题问 Claude，答案口语化、适合朗读。
- **绑定技能**：可以把某个触发词绑到一个 Claude 技能上。比如说「问问贾维斯，特斯拉」就自动跑期权扫描技能，把结论念给你听。

> ⚠️ 小爱会先用它自己的逻辑应一句（因为本应用读的是事后的云端对话记录，不是麦克风拦截），约几秒后本应用再用它的声音念出 Claude 的答案——属于固有延迟，非实时。

## 依赖

- Node.js ≥ 20（`npm run dev` 免打包模式需要 ≥ 23）
- 已安装并能正常使用的 Claude Code CLI（`claude`）
- 电脑和小爱音箱登录**同一个小米账号**

## 安装与运行

```bash
git clone <repo-url> xiaoai-cc
cd xiaoai-cc
npm install

npm run build   # 用 esbuild 打包成 dist/app.js
npm start       # 运行
```

或免打包直接从源码跑（Node ≥ 23）：

```bash
npm run dev
```

## 首次使用

1. 启动后会打印一个**小米登录二维码链接**，macOS 上还会自动打开二维码图片；
2. 用手机「米家」或「小米」App 扫码确认登录；
3. 登录态会存到 `data/store.json`，**之后启动自动恢复，无需再扫码**；
4. 账号下有多台音箱时让你选一台，只有一台则自动选中；
5. 看到 `就绪` 后，对小爱说：**「问问克劳德，<你的问题>」**。

## 配置（可选）

不改代码、不用重新打包：把 `config.example.json` 复制为 `config.local.json`，改完重启即可生效（只覆盖你写到的字段，其余用内置默认值）。`config.local.json` 已被 `.gitignore` 忽略，不会上传。

```json
{
  "TRIGGERS": [
    { "keywords": ["问问贾维斯", "特斯拉"], "skill": "news" },
    { "keywords": ["问问克劳德", "帮我问"] }
  ],
  "CLAUDE_MODEL": "haiku",
  "POLL_INTERVAL_MS": 1000
}
```

| 字段 | 说明 |
|---|---|
| `TRIGGERS` | 触发规则。每条 = 一组触发词 `keywords` + 可选 `skill`。命中触发词后，后面的文字当参数/问题；带 `skill` 就直接跑该技能，不带就是普通问答。匹配取最具体（更长）的触发词 |
| `CLAUDE_MODEL` | 模型别名 `haiku` / `sonnet` / `opus` 或完整模型 ID。默认 `haiku`（语音问答又快又省） |
| `POLL_INTERVAL_MS` | 对话轮询间隔（毫秒），默认 1000 |

## 数据与隐私

- 登录凭证（serviceToken / passToken）只存在本地 `data/store.json`，已被 `.gitignore` 忽略，切勿提交；
- 删除 `data/` 即退出登录、清空设备配置；
- 全程只和小米官方云 + 本机 `claude` 通信，无第三方上报。
